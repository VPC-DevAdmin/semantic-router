Model tiers

The model ladder, cheapest first. Every prompt is scored for difficulty and sent to the lowest tier that clears its bar — easy prompts stay cheap, only hard ones reach the frontier. Saved to config/live_demo.local.json; the benchmark config is never touched.

Routing logic

Decide which prompts are worth a bigger model. Each signal recognises a kind of work; raising its influence pushes matching prompts toward pricier tiers (cheap signals push the other way). The cutoffs set where one tier hands off to the next. Changes apply to the live router.
